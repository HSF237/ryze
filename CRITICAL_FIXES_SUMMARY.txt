FIX #2: CART BUG (FIX SECOND)
================================================================================

Problem: Cart drawer stuck open, nothing clickable
Status: ✅ FIXED LOCALLY
Impact: Cart drawer overlays entire page, blocking navigation

What was wrong:
  - State boolean `open: false` was OVERWRITTEN by function `open: () => ...`
  - CartDrawer was checking `cart.open` expecting a boolean, but got a function
  - Functions are always truthy, so condition `cart.open ?` was always true
  - Cart drawer stayed visible permanently

How to fix:
  Changed property name: `state.open` → `isOpen`
  - CartDrawer now checks `cart.isOpen` (boolean)
  - Navbar still uses `cart.open()` (function) for onClick
  - No naming conflict anymore

How to push:
  Run: FIX_CART_BUG.bat

Files changed:
  • lib/cart.jsx (line 60: open → isOpen, removed duplicate properties)
  • components/CartDrawer.jsx (lines 16, 23: cart.open → cart.isOpen)

Expected result:
  ✓ Cart drawer opens only when clicking cart button
  ✓ Cart drawer closes when clicking X or backdrop
  ✓ Rest of page is interactive
  ✓ Cart can be clicked open/closed freely
