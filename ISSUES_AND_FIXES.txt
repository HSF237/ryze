ISSUE #2: CART BUG - CART DRAWER STUCK OPEN
================================================================================

🔴 PROBLEM:
   When opening https://ryze-mu.vercel.app, the cart drawer was:
   - Already open on page load
   - Could not be closed
   - Blocking all interaction with the rest of the page
   - Only scrolling worked (because it was modal overlay)

ROOT CAUSE:
   Naming conflict in lib/cart.jsx context value:

   const initial = { items: [], open: false }  // Boolean state

   const value = {
     ...state,  // Spreads open: false
     open: () => dispatch({ type: 'open' }),  // OVERWRITES with function
     close: () => dispatch({ type: 'close' })
   }

   Result: cart.open became a function, not a boolean

   CartDrawer expected:
   ```javascript
   className={`... ${cart.open ? 'opacity-100' : 'opacity-0'}`}
   //                    ^^^^^^^^^ Expected boolean
   ```

   But got:
   ```javascript
   cart.open = () => ...  // A function
   cart.open ? ... : ...  // Always truthy (functions are truthy)
   // Result: Always shows 'opacity-100' → always visible
   ```

IMPACT:
   - Cart drawer opened immediately on page load
   - Couldn't close it
   - Couldn't interact with anything else
   - User experience completely broken

✅ FIX:
   Separated the state boolean from the action function:

   lib/cart.jsx:
   - Changed: open: state.open → isOpen: state.open
   - Kept: open: () => dispatch(...) for Navbar's onClick handler
   - Kept: close: () => dispatch(...) for close button

   CartDrawer.jsx:
   - Changed: cart.open (function) → cart.isOpen (boolean)
   - Line 16: Updated backdrop className check
   - Line 23: Updated drawer className check

   Now:
   - cart.isOpen = false (boolean for drawer visibility)
   - cart.open() = function for opening cart
   - cart.close() = function for closing cart
   - No naming conflict!

DEPLOY METHOD:
   Run: FIX_CART_BUG.bat

EXPECTED RESULT:
   ✓ Cart closed on page load

   ✓ Backdrop click closes drawer
   ✓ Page fully interactive
